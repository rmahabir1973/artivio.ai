POST
/api/v1/jobs/createTask
Create Task
Create a new generation task

Request Parameters
The API accepts a JSON payload with the following structure:

Request Body Structure
{
  "model": "string",
  "callBackUrl": "string (optional)",
  "input": {
    // Input parameters based on form configuration
  }
}
Root Level Parameters
model
Required
string
The model name to use for generation

Example:

"bytedance/v1-lite-text-to-video"
callBackUrl
Optional
string
Callback URL for task completion notifications. Optional parameter. If provided, the system will send POST requests to this URL when the task completes (success or failure). If not provided, no callback notifications will be sent.

Example:

"https://your-domain.com/api/callback"
Input Object Parameters
The input object contains the following parameters based on the form configuration:

input.prompt
Required
string
The text prompt used to generate the video

Max length: 10000 characters
Example:

"Wide-angle shot: A serene sailing boat gently sways in the harbor at dawn, surrounded by soft Impressionist hues of pink and orange with ivory accents. The camera slowly pans across the scene, capturing the delicate reflections on the water and the intricate details of the boat's sails as the light gradually brightens."
input.aspect_ratio

Optional
boolean
Whether to fix the camera position

Boolean value (true/false)

input.seed
Optional
integer
Random seed to control video generation. Use -1 for random.

input.enable_safety_checker
Optional
boolean
The safety checker is always enabled in Playground. It can only be disabled by setting false through the API.

Boolean value (true/false)
Example:

  -H "Authorization: Bearer YOUR_API_KEY" \
  -d '{
    "model": "bytedance/v1-lite-text-to-video",

Response Example
{
  "code": 200,
  "message": "success",
  "data": {
    "taskId": "task_12345678"
  }
}
Response Fields
code
Status code, 200 for success, others for failure
message
Response message, error description when failed
data.taskId
Task ID for querying task status
